# Setup And Architecture

Use this file for new-site scaffolding or integrating Fumadocs into an existing app.

## Choose The Entry Path

- New docs app: start from Quick Start or `create-fumadocs-app`.
- Existing Next.js app: use manual installation instead of copying starter output blindly.

## Quick Start Facts

- Official quick start currently requires Node.js `22`.
- `npm create fumadocs-app` is the standard scaffold path.
- The current scaffold family supports `Next.js`, `Waku`, `React Router`, and `Tanstack Start`.
- Quick Start says new apps come with LLM integration and dynamic metadata image configured by default; verify generated output before claiming exact files.

## `create-fumadocs-app`

- Official page: https://www.fumadocs.dev/docs/cli/create-fumadocs-app
- The package can be used as a CLI or imported from scripts.
- The docs example uses:

```ts
import { create } from 'create-fumadocs-app';

await create({
  outputDir: 'my-app',
  template: '+next+fuma-docs-mdx',
  packageManager: 'pnpm',
});
```

Use the official scaffold when the user wants a fresh Fumadocs app or a standalone docs site.

## Manual Next.js Integration Baseline

Official page: https://www.fumadocs.dev/docs/manual-installation/next

Current prerequisites called out by the docs:

- `Next.js 16`
- `Tailwind CSS 4`

## Required Building Blocks

### 1. Content Source

Install the MDX/content-source packages:

```bash
npm i fumadocs-mdx fumadocs-core @types/mdx
```

Create `source.config.ts`:

```ts
import { defineDocs, defineConfig } from 'fumadocs-mdx/config';

export const docs = defineDocs({
  dir: 'content/docs',
});

export default defineConfig();
```

### 2. Next Config

- Use `createMDX` from `fumadocs-mdx/next`.
- Prefer `next.config.mjs`; the docs note that `fumadocs-mdx` is ESM-only.
- If the repo uses a TypeScript Next config, verify native TS resolver support instead of assuming it works.

### 3. Loader

Create `lib/source.ts` from the generated collection output:

```ts
import { docs } from 'collections/server';
import { loader } from 'fumadocs-core/source';

export const source = loader({
  baseUrl: '/docs',
  source: docs.toFumadocsSource(),
});
```

- `.source` is generated by `next dev` or `next build`.
- `loader()` becomes the truth source for page lookup, page tree, and static params.

### 4. Root Layout

- Put `RootProvider` at the root layout.
- Add the required `body` class when following the official layout baseline:

```tsx
<body className="flex flex-col min-h-screen">
  <RootProvider>{children}</RootProvider>
</body>
```

### 5. Styles

The baseline theme imports are:

```css
@import 'tailwindcss';
@import 'fumadocs-ui/css/neutral.css';
@import 'fumadocs-ui/css/preset.css';
```

- Fumadocs does not impose a default font; use `next/font` or repo-native font setup.

### 6. Shared Layout Options

The docs recommend a shared helper like `lib/layout.shared.tsx` for nav/layout options:

```ts
import type { BaseLayoutProps } from 'fumadocs-ui/layouts/shared';

export function baseOptions(): BaseLayoutProps {
  return {
    nav: {
      title: 'My App',
    },
  };
}
```

### 7. Route Surface

Expect to wire these seams in some form:

- `components/mdx.tsx`
- `app/docs/layout.tsx`
- `app/docs/[[...slug]]/page.tsx`
- `app/api/search/route.ts`

Do not assume starter file names if the repo already adapted them.
